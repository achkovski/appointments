# 1. БАРАЊА ОД КЛИЕНТИТЕ

## 1.1 Вовед и опис на проблемот

### Контекст и мотивација

Како сопственик на бизнис кој нуди услуги базирани на закажување термини, се соочувам со значителни предизвици во секојдневното управување со резервации. Традиционалните методи на закажување - телефонски повици, рачно водење на евиденција во тетратки или едноставни табели - станаа неефикасни и подложни на грешки.

### Опис на проблемот

Главните проблеми со кои се соочувам се:

**1. Неефикасна комуникација**
- Постојани телефонски прекини во текот на работното време
- Клиентите не можат да резервираат надвор од работно време
- Губење време на повторувачки телефонски разговори
- Недостапност за клиенти кои претпочитаат дигитална комуникација

**2. Рачно управување со термини**
- Ризик од двојни резервации (double booking)
- Тешко следење на слободни термини
- Неможност за брзо прилагодување на распоредот
- Човечки грешки при внесување на податоци

**3. Недостаток на организација**
- Без централизирана база на клиенти
- Без историја на минати резервации
- Тешко планирање на капацитети
- Неможност за анализа на бизнис трендови

**4. Лоша клиентска искуство**
- Клиентите мора да чекаат за потврда
- Нема автоматски потсетници за термини
- Непрофесионален впечаток
- Отежнато откажување или промена на термини

### Потреба за решение

Потребен ми е модерен, дигитален систем кој ќе:

1. **Автоматизира процесот на резервација** - Клиентите самостојно да можат да резервираат термини 24/7 без моја интервенција

2. **Елиминира конфликти** - Системот автоматски да спречува двојни резервации и да ажурира достапност во реално време

3. **Централизира податоци** - Сите резервации, податоци за клиенти и историја на едно место

4. **Подобри комуникација** - Автоматски емаил потврди, потсетници и известувања

5. **Овозможи аналитика** - Увид во бизнис перформанси, популарни термини и трендови

6. **Биде достапен** - Уникатна веб страница за мојот бизнис која лесно се споделува

### Целни корисници

Системот треба да опслужува два типа на корисници:

**Бизнис корисници (јас и слични претприемачи):**
- Индивидуални даватели на услуги (фризери, масери, консултанти, фитнес тренери)
- Мали бизниси (стоматолошки ординации, медицински кабинети, козметички салони)
- Средни бизниси (ресторани, хотели, агенции)

**Клиенти (крајни корисници):**
- Лица кои сакаат да резервираат термин онлајн
- Корисници кои претпочитаат дигитална наспроти телефонска комуникација
- Клиенти кои сакаат флексибилност (резервација во било кое време)

### Очекувани придобивки

Со имплементација на ваков систем очекувам:

| Придобивка | Опис |
|------------|------|
| Заштеда на време | Намалување на телефонски повици за 70-80% |
| Зголемени резервации | Достапност 24/7 = повеќе клиенти |
| Помалку грешки | Автоматизација = нула двојни резервации |
| Подобра организација | Јасен преглед на дневен/неделен/месечен распоред |
| Професионален имиџ | Модерен дигитален пристап кон клиентите |
| Информирани одлуки | Аналитика за оптимизација на работното време |

# 2. СПЕЦИФИКАЦИИ

## 2.1 Преглед на системот

### Опис на системот

Системот за онлајн резервација на термини (Appointments) е веб апликација која овозможува бизниси и индивидуални даватели на услуги да управуваат со своите резервации дигитално. Системот се состои од три главни компоненти:

1. **Бизнис Dashboard** - Интерфејс за бизнис корисниците за управување со сите аспекти на резервациите
2. **Јавна Booking Страница** - Интерфејс за клиентите за правење резервации
3. **Backend API** - Серверска логика која ги поврзува сите компоненти

## 2.2 Спецификација на функционалности

### 2.2.1 Автентикација и авторизација

#### Регистрација на корисник

**Опис:** Нов бизнис корисник креира сметка во системот.

**Влезни податоци:**
- Емаил адреса (валиден формат, уникатен)
- Лозинка (минимум 8 карактери)
- Име
- Презиме

**Процес:**
1. Корисникот го пополнува регистрацискиот формулар
2. Frontend валидација на полињата
3. Backend валидација (уникатност на емаил, јачина на лозинка)
4. Хеширање на лозинката со bcrypt (10 rounds)
5. Креирање на корисник во базата со `email_verified = false`
6. Генерирање на verification token
7. Испраќање на верификациски емаил
8. Прикажување на порака за успешна регистрација

**Излез:**
- Порака: "Регистрацијата е успешна. Проверете го вашиот емаил за верификација."
- Емаил со верификациски линк

**Грешки:**
- "Емаилот веќе постои во системот"
- "Лозинката мора да има минимум 8 карактери"
- "Невалиден формат на емаил"

#### Верификација на емаил

**Опис:** Корисникот ја потврдува својата емаил адреса.

**Влез:** Verification token од URL

**Процес:**
1. Корисникот кликнува на линкот од емаилот
2. Backend го валидира токенот
3. Ако е валиден: `email_verified = true`, се брише токенот
4. Редирекција на login страница со success порака

**Грешки:**
- "Невалиден или истечен верификациски линк"

#### Најава

**Опис:** Регистриран корисник се најавува во системот.

**Влезни податоци:**
- Емаил
- Лозинка

**Процес:**
1. Валидација на credentials
2. Проверка дали емаилот е верифициран
3. Споредба на хеш на лозинката
4. Генерирање на JWT токен (7 дена валидност)
5. Враќање на токен и корисничка информација

**Излез:**
- JWT токен
- Корисничка информација (id, email, firstName, lastName, role)
- hasCompletedSetup flag (дали има бизнис профил)

**Грешки:**
- "Невалидни credentials"
- "Емаилот не е верифициран"

### 2.2.5 Slot Calculator - Алгоритам за пресметка на слотови

**Влез:**
- businessId
- date (датум за кој се бараат слотови)
- serviceId (за да се добие времетраење)

**Алгоритам:**

```
1. Добиј работно време за денот (day_of_week)
   - Ако is_available = false → врати празна листа

2. Провери за специјален датум
   - Ако постои и is_available = false → врати празна листа
   - Ако постои со изменето време → користи го тоа време

3. Добиј паузи за денот

4. Добиј времетраење на услугата (duration)

5. Добиј постоечки резервации за датумот

6. Генерирај слотови:
   slots = []
   current_time = start_time

   WHILE current_time + duration <= end_time:
       slot_end = current_time + duration

       // Провери дали слотот паѓа во пауза
       IF NOT overlaps_with_break(current_time, slot_end, breaks):

           // Провери дали слотот е резервиран
           IF NOT is_booked(current_time, slot_end, appointments):
               slots.add({
                   start: current_time,
                   end: slot_end,
                   available: true
               })
           ELSE:
               // За multiple mode, провери капацитет
               IF capacity_mode == 'multiple':
                   bookings_count = count_bookings(current_time, appointments)
                   IF bookings_count < max_capacity:
                       slots.add({start, end, available: true, remaining: max_capacity - bookings_count})

       current_time = current_time + duration

7. Врати листа на слотови
```

**Излез:**
```json
{
  "date": "2024-12-20",
  "slots": [
    { "start": "09:00", "end": "09:30", "available": true },
    { "start": "09:30", "end": "10:00", "available": true },
    { "start": "10:00", "end": "10:30", "available": false },
    ...
  ]
}
```

## 2.6 Бизнис правила и валидација

### Валидациски правила

| Ентитет | Правило |
|---------|---------|
| Email | Валиден формат, уникатен |
| Лозинка | Мин. 8 карактери |
| Бизнис име | Мин. 2, макс. 255 карактери |
| Slug | Автоматски од име, уникатен |
| Услуга | Име задолжително, времетраење > 0 |
| Работно време | end_time > start_time |
| Пауза | Во рамки на работно време |
| Резервација | Слотот мора да е достапен |

### Бизнис логика

1. **Двојна резервација** - Не е дозволена за single mode
2. **Капацитет** - Автоматско блокирање кога е полн
3. **Емаил потврда** - Ако е вклучена, статус останува pending
4. **Откажување** - Испраќа нотификација до клиент
5. **Минати термини** - Не може да се уредуваат
